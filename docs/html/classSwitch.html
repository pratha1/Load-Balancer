Detailed Description

Routes requests between multiple LoadBalancers.

The Switch:

  • Generates random traffic
  • Creates burst traffic
  • Distributes requests by type
  • Runs simulation over time

Runs the full simulation.

Runs the full simulation for a specified duration.

Parameters
durationNumber of clock cycles to simulate.

Simulation Steps:

  1. Generate an initial queue load
  2. For each clock cycle:
    • Generate burst traffic
    • Route requests based on job type
    • Advance both load balancers
  3. Print final summaries
